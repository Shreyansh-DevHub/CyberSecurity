# Project 5: Network Monitoring & Troubleshooting Lab

## Overview
This project was done on Windows to practice basic network monitoring and troubleshooting.  
The goal was to learn how to capture traffic, check open connections, and test connectivity using built‑in tools and Wireshark.

---

## Steps

### 1. Wireshark Capture
- Ran Wireshark while using `ping 8.8.8.8`.
- Captured ICMP request and reply packets.
- Saved capture in respective folder

### 2. Netstat
- Used `netstat -ano` in Command Prompt.
- Showed active connections and listening ports.
- Saved output in respective folder

### 3. Ping & Tracert
- Ran `ping 8.8.8.8` to check connectivity.
- Ran `tracert 8.8.8.8` to see the path to Google DNS.
- Saved results in respective folder
---

## Learning Points
- Wireshark shows detailed packet information.
- Netstat helps find open ports and active connections.
- Ping and tracert confirm connectivity and show network paths.
- These tools are useful for troubleshooting and understanding how networks work.

---

## Conclusion
This lab gave hands‑on practice with basic monitoring tools on Windows.  
It helped connect theory from networking classes with real commands and captures.
