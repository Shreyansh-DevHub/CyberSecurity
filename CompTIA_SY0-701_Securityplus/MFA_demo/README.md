# Project 7: Multi-Factor Authentication (MFA) Simulation

## Overview
This project demonstrates how multi-factor authentication (MFA) enhances account security by requiring two verification steps — a password and a one-time code (OTP).

---

## Steps
1. User enters password.
2. If correct, system generates a random 6-digit OTP.
3. User enters OTP to complete authentication.
4. Access is granted only if both factors match.

---

## Learning Points
- MFA adds an extra layer of protection beyond passwords.
- Even if a password is compromised, the attacker still needs the OTP.
- Demonstrates the principle of **defense in depth** in identity security.

---

## Conclusion
This simulation shows how MFA prevents unauthorized access and strengthens authentication systems.  
It completes the Security+ lab series with a practical demonstration of modern identity protection.
